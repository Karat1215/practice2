# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | Абстрактный класс | Есть общее состояние (имя, возраст) — это отношение "является". Абстрактный класс позволяет хранить поля и общую логику. |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | Интерфейс | Это способность ("умеет"), объекты из разных иерархий. Интерфейс поддерживает множественную реализацию. |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | Абстрактный класс | Есть общее состояние (скорость, топливо) и отношение "является" транспортным средством. |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | Интерфейс | Это способность ("умеет"), объекты из разных иерархий. Интерфейс позволяет добавить общее поведение. |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс | Есть общее состояние (цвет) и отношение "является" фигурой. Можно вынести общие поля и методы. |
| Класс должен одновременно уметь летать, плавать и ездить | Интерфейсы | Нужна множественная реализация. Java не поддерживает множественное наследование классов. Создать три интерфейса: Flyable, Swimmable, Drivable. |
